Git Branch Workflow

Updating your Main Branch from a Feature Branch

Updating your main branch with changes from a feature branch (like AI) is a standard part of the Git workflow. Depending on whether you want a clean history or a detailed one, you have two primary options.

Option 1: The Standard Merge

This is the most common method. It preserves the history of your AI branch and adds a "merge commit" to the main branch.

  1. Commit your changes on the AI branch: 
    git add .
git commit -m "Describe your changes here"
  2. Switch to the main branch: 
    git checkout main
  3. Pull the latest changes (always good practice to avoid conflicts): 
    git pull origin main
  4. Merge the AI branch: 
    git merge AI
  5. Push to the remote server: 
    git push origin main

Option 2: The Rebase (For a Linear History)

If you want your commit history to look like a straight line without "merge bubbles," you can use rebase. This effectively moves your AI commits to the tip of the main branch.

  1. While on the AI branch, pull the latest from main: 
    git fetch origin
git rebase origin/main
  2. Switch to main: 
    git checkout main
  3. Merge (it will be a "fast-forward"): 
    git merge AI
  4. Push: 
    git push origin main

Which one should you choose?

Feature Merge Rebase
History Traceable, shows exactly when branches joined. Clean, linear, and easy to read.
Complexity Simple and safe. Can be tricky if multiple people work on the same branch.
Best For Public projects and shared branches. Personal cleanup before pushing to a shared repo.
Pro Tip: If you encounter a "merge conflict," Git will pause and ask you to pick which code to keep. After you manually fix the files, you'll need to git add them and run git commit to finish the merge.
